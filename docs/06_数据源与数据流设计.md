# 数据源与数据流设计

## 6.1 数据管道架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   数据源     │────→│  数据适配器  │────→│  数据转换层  │────→│   图表组件   │
│ (API/JSON)  │     │ (统一格式)   │     │ (过滤/聚合)  │     │ (可视化渲染) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                           ↑                                    ↓
                    ┌─────────────┐                    ┌─────────────┐
                    │  缓存层      │←───────────────────│  交互事件    │
                    │ (IndexedDB)  │    用户筛选/钻取     │ (点击/缩放)  │
                    └─────────────┘                    └─────────────┘
```

## 6.2 统一数据格式 (UDF)

所有数据源最终转换为标准格式：

```typescript
interface UnifiedDataset {
  meta: {
    sourceId: string;
    updateTime: string;
    totalCount?: number;
  };
  dimensions: Array<{
    name: string;      // 字段名
    type: 'string' | 'number' | 'date' | 'boolean';
    role: 'category' | 'series' | 'value' | 'time';  // 图表角色
  }>;
  data: Array<Record<string, any>>;  // 标准行数据
}
```

## 6.3 实时同步策略

| 场景               | 策略                                               |
| :----------------- | :------------------------------------------------- |
| **多图表同数据源** | 一次请求，共享缓存，各图表独立筛选转换             |
| **定时刷新**       | 全局设置默认间隔（5min），单图表可覆盖             |
| **数据更新**       | WebSocket 推送差异数据，局部更新图表（不整页刷新） |
| **离线状态**       | 显示最后更新时间，数据旁加"离线"灰色标签           |
